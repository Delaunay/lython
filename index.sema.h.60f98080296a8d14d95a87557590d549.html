26 /* The semantic analysis (SEM-A) happens after the parsing, the AST can be assumed to be
27 * syntactically correct its job is to detect issues that could prevent a succesful compilation.
28 *
29 * Errors caught in that process are undeclared variables and mistypings,
30 * this includes missing attributes, missing methods
31 *
32 * In addition, our SEM-A will deduce types (i.e variables inherit the type of the expressions,
33 * this is NOT type inference) and allocate a register to each variables.
34 *
35 * To support type deduction SEM-A returns the type of the analysed expression,
36 * the deduction can then be used for typechecking.
37 *
38 * Type deduction is a weaker form of type inference where the type of the parent parent
39 * expression is deduced from the children. In the future we might add full type inference. Type
40 * deduction will still be useful then as it will reduce the cost of type inference for the
41 * trivial cases.
42 *
43 * Type deduction alone should provide a satisfactory development experience, as the user should
44 * only have to specify the type of the arguments which is good practice anyway as it serves as
45 * documentation.
46 *
47 * Raises
48 * ------
49 *
50 * TypeError
51 * when types between expression mismatch
52 *
53 * AttributeError
54 * When using an object attribute that does not exist
55 *
56 * NameError
57 * When using an undefined variable
58 *
59 */
