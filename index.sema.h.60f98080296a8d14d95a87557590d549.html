183 /* The semantic analysis (SEM-A) happens after the parsing, the AST can be assumed to be
184 * syntactically correct its job is to detect issues that could prevent a succesful compilation.
185 *
186 * Errors caught in that process are undeclared variables and mistypings,
187 * this includes missing attributes, missing methods
188 *
189 * In addition, our SEM-A will deduce types (i.e variables inherit the type of the expressions,
190 * this is NOT type inference) and allocate a register to each variables.
191 *
192 * To support type deduction SEM-A returns the type of the analysed expression,
193 * the deduction can then be used for typechecking.
194 *
195 * Type deduction is a weaker form of type inference where the type of the parent parent
196 * expression is deduced from the children. In the future we might add full type inference. Type
197 * deduction will still be useful then as it will reduce the cost of type inference for the
198 * trivial cases.
199 *
200 * Type deduction alone should provide a satisfactory development experience, as the user should
201 * only have to specify the type of the arguments which is good practice anyway as it serves as
202 * documentation.
203 */
