28 /* The semantic analysis (SEM-A) happens after the parsing, the AST can be assumed to be
29 * syntactically correct its job is to detect issues that could prevent a succesful compilation.
30 *
31 * Errors caught in that process are undeclared variables and mistypings,
32 * this includes missing attributes, missing methods
33 *
34 * In addition, our SEM-A will deduce types (i.e variables inherit the type of the expressions,
35 * this is NOT type inference) and allocate a register to each variables.
36 *
37 * To support type deduction SEM-A returns the type of the analysed expression,
38 * the deduction can then be used for typechecking.
39 *
40 * Type deduction is a weaker form of type inference where the type of the parent parent
41 * expression is deduced from the children. In the future we might add full type inference. Type
42 * deduction will still be useful then as it will reduce the cost of type inference for the
43 * trivial cases.
44 *
45 * Type deduction alone should provide a satisfactory development experience, as the user should
46 * only have to specify the type of the arguments which is good practice anyway as it serves as
47 * documentation.
48 *
49 * Notes
50 * -----
51 *
52 * SEM-A does a quick first pass through the module to insert definitions
53 * that are used before their definitions. This allow us to get away
54 * with not forward-declaring everything, but it means some analytics
55 * will get delayed until the end. In the case of mutually recursive definitions
56 * forward declaration is required so typing can be checked.
57 *
58 * SEM-A will add type annotation & reorder arguments wherever it can.
59 * This has the goal and standardizing the code & simplifying its execution
60 * later on.
61 *
62 * You can inspect the change by saving the resulting AST.
63 * You could implement an automatic formatter that executes semantic analysis
64 * to format & complete the code. The completed code will then take less time
65 * to compile as the representation will be easier to analyse.
66 *
67 * Raises
68 * ------
69 *
70 * TypeError
71 * Raised when types between expression mismatch
72 *
73 * UnsupportedOperand
74 * Raised when using an operand on an object that does not support it
75 *
76 * AttributeError
77 * Raised when using an object attribute that does not exist
78 *
79 * NameError
80 * Raised when using an undefined variable
81 *
82 * ModuleNotFoundError
83 * Raised when importing a module that was not found
84 *
85 * ImportError
86 * Raised when importing a statement that was not found from a module
87 *
88 */
