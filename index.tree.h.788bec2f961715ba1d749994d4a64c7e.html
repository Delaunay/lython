32 /* Tree evaluator is a very simple interpreter that is also very slow.
33 * It takes as input the binding array generated by the semantic analysis
34 * i.e the evaluation context and the expression to evaluate given the context
35 *
36 * The expression to evaluate is often a call to a function, for a standard
37 * program that function will be main.
38 *
39 * While being slow this evaluator has the advantage or returning an AST
40 * as result which makes it perfect for compile-time usage.
41 *
42 * We can call the evaluator on standard declaration which will result in
43 * constant folding everything it can. Note that because we are able to represent
44 * complex types at compile time, string/list/dict even object operations can be folded.
45 *
46 * .. code-block:: python
47 *
48 * value = '.'.join(['a', 'b', 'c']) # <= Everything is known at compile time
49 * # It can be folded
50 *
51 * Additionally, this can be used to generate code at compile time by
52 * creating function with types as arguments.
53 *
54 * .. code-block::
55 *
56 * def Point(type: Type):
57 * class point:
58 * x: type
59 * y: type
60 *
61 * return point
62 *
63 * Pointi = Point(int)
64 * Pointf = Point(float) # <= Generate new types at compile time
65 *
66 *
67 * Evaluation Implementation
68 * -------------------------
69 *
70 * 1. Reuse as much as possible from the sema context
71 * Save the context inside each statement so we can use it
72 * during evaluation.
73 * Because the context is copied, it is easy to do parallel executions
74 *
75 * 2. Create a different context for evaluation only
76 */
