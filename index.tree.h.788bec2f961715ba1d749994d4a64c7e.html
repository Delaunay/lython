24 /* Tree evaluator is a very simple interpreter that is also very slow.
25 * It takes as input the binding array generated by the semantic analysis
26 * i.e the evaluation context and the expression to evaluate given the context
27 *
28 * The expression to evaluate is often a call to a function, for a standard
29 * program that function will be main.
30 *
31 * While being slow this evaluator has the advantage or returning an AST
32 * as result which makes it perfect for compile-time usage.
33 *
34 * We can call the evaluator on standard declaration which will result in
35 * constant folding everything it can. Note that because we are able to represent
36 * complex types at compile time, string/list/dict even object operations can be folded.
37 *
38 * .. code-block:: python
39 *
40 * value = '.'.join(['a', 'b', 'c']) # <= Everything is known at compile time
41 * # It can be folded
42 *
43 * Additionally, this can be used to generate code at compile time by
44 * creating function with types as arguments.
45 *
46 *
47 * .. code-block::
48 *
49 * def Point(type: Type):
50 * class point:
51 * x: type
52 * y: type
53 *
54 * return point
55 *
56 * Pointi = Point(int)
57 * Pointf = Point(float) # <= Generate new types at compile time
58 *
59 *
60 * Evaluation Implementation
61 * -------------------------
62 *
63 * 1. Reuse as much as possible from the sema context
64 * Save the context inside each statement so we can use it
65 * during evaluation.
66 * Because the context is copied, it is easy to do parallel executions
67 *
68 * 2. Create a different context for evaluation only
69 */
